# Variants

Use this reference when the user asks for options, alternatives, explorations, A/B/C, or multiple mockups.

## Variation Depth

Tight variation:

- Use when the user says only one thing should vary.
- Change exactly that dimension.
- Keep structure, data, palette, typography, device, and surrounding context fixed.

Moderate variation:

- Use as the default for "show options" or "visualize a few directions".
- Keep product identity and key content fixed.
- Vary layout grouping, emphasis, density, navigation treatment, or control model.

Wide variation:

- Use only when the user asks to reimagine, depart from the current design, or explore very different concepts.
- Vary structure, metaphor, density, motion implication, and visual language.
- Keep product semantics, core task, and required content intact.

## Variant Count

- "A/B" means two options.
- "A/B/C", "a few", "several", or unspecified options means three options.
- More than three usually reduces inspection quality; ask only if the user's request truly needs many.

## Labeling

- Label options A, B, and C outside the UI.
- Add short option names only if they help: "A. Ledger", "B. Focus", "C. Timeline".
- Do not place option labels inside app navigation or user-facing product copy.

## Prompt Pattern

For each variant, define:

- Thesis: one short sentence.
- Vary: the design dimension allowed to change.
- Preserve: platform, product identity, required content, exact text, palette, and constraints.
- Best for: the product decision this option helps answer.

## Good Variation Axes

- Information density.
- Navigation depth.
- Action placement.
- Summary-first vs ledger-first.
- Inline editing vs staged confirmation.
- Visual emphasis on current state vs future state.
- Component shape: row, strip, sheet, timeline, control cluster, inspector.

## Poor Variation Axes

- Random palette changes.
- More decorative chrome.
- Different fake data in every option.
- Different product names.
- Arbitrary icon style changes.
- Generic premium vs more generic premium.
