# Research foundations (web-verified)

Use this file when you need concrete thresholds, not just style preference.

## Accessibility and UI clarity
- WCAG 2.2 Contrast (Minimum): text contrast baseline guidance.
  - https://www.w3.org/WAI/WCAG22/Understanding/contrast-minimum.html
- WCAG 2.2 Non-text Contrast: 3:1 guidance for UI components and state indicators.
  - https://www.w3.org/WAI/WCAG22/Understanding/non-text-contrast.html
- WCAG 2.2 Target Size (Minimum): pointer targets and spacing exceptions.
  - https://www.w3.org/WAI/WCAG22/Understanding/target-size-minimum.html
- WCAG 2.2 Focus Not Obscured (Minimum): keep focused elements visible.
  - https://www.w3.org/WAI/WCAG22/Understanding/focus-not-obscured-minimum.html
- WCAG Headings and Labels: descriptive, purpose-driven labels.
  - https://www.w3.org/WAI/WCAG22/Understanding/headings-and-labels
- WCAG Text Spacing: content should remain usable with increased spacing.
  - https://www.w3.org/WAI/WCAG21/Understanding/text-spacing

## Forms and input UX
- WAI Forms tutorial: labels, instructions, and guidance.
  - https://www.w3.org/WAI/tutorials/forms/instructions/
- WAI Forms notifications: feedback, validation, and message design.
  - https://www.w3.org/WAI/tutorials/forms/notifications/

## Motion and interaction ergonomics
- MDN prefers-reduced-motion media feature.
  - https://developer.mozilla.org/en-US/docs/Web/CSS/%40media/prefers-reduced-motion
- Apple accessibility guidance for reducing motion on iPhone.
  - https://support.apple.com/en-lb/guide/accessibility-for-iphone/iph0b691d3ed/ios
- web.dev interaction design (hover/pointer media query behavior).
  - https://web.dev/learn/design/interaction
