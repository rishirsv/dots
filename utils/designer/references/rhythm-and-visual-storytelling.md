# Rhythm and visual storytelling

Use this for page/flow cohesion and reading order.

## Reading order blueprint
Each major section should include:
- a clear headline
- one sentence of why it matters
- proof/details (optional)
- a next action (optional)

If a section feels confusing, tighten this order before changing visual style.

## Density and pacing
- Alternate dense regions and breathable regions to reduce fatigue.
- Keep section spacing predictable so users can build scanning rhythm.
- Use transitions consistently between section types (hero, feature, proof, CTA).

Quick checks:
- Do two dense sections stack without a visual "breath"?
- Are section top/bottom spacings from the same scale?
- Does each section feel intentionally short, medium, or long?

## Grouping and scannability
- Prefer whitespace + alignment over decorative containers.
- Use subheads as scan anchors for long pages.
- Turn heavy paragraphs into short blocks, bullets, or callouts when clarity improves.

Quick checks:
- Can users find key info by scanning subheads only?
- Are related items visually closer than unrelated ones?
- Is there any "wall of text" that should be chunked?

## Consistent system behavior
- Buttons, cards, and inputs should behave like one system:
  - same radius families
  - same spacing families
  - same focus and disabled treatment
- Reuse component behavior patterns across sections.

Quick checks:
- Do equal-priority actions look and behave the same?
- Are similar components surprisingly different in spacing or states?
- Is interaction feedback consistent across the page?
