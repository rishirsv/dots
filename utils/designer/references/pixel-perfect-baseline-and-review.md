# Pixel-perfect baseline and review

Use this for iterative redesign of existing UI.

## Baseline-first rule

Before proposing design changes:
- Create a faithful, pixel-accurate baseline of the current UI.
- Base the baseline on real repo code and context files.

## Required inputs

Use `.design/components.md`, `.design/layouts.md`, `.design/routes.md`, and `.design/theme.md`, plus relevant page-level source files.

## Review-before-presenting rule

Before presenting design recommendations:
- Read the current design output/state.
- Verify it still matches the source-informed baseline.
- Then present findings and proposed improvements.

## Iteration behavior

- Apply changes in small, deliberate passes.
- Keep recommendations scoped to the requested surface.
- Do not require branch-variant generation.
