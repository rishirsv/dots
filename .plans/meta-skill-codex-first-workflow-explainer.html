V1 is a batch runner, not an archive viewer.

The parent Codex thread launches many child threads/worktrees, reads their structured results, and reports progress. Files are only recovery state.

1. Load tasks

Parent gets 30 tasks from evals or improve lane.

2. Launch children

Starts child threads with bounded concurrency.

3. Children work

Each child has its own worktree.

4. Read results

Parent reads structured child finals.

5. Report status

Parent shows completed/running/blocked/errored.

What The User Sees

30tasks launched
12completed
16running
2blocked

The parent thread is the status surface. The user opens a child thread or worktree only when a result needs inspection or follow-up.

Runtime Shape

Parent thread

Owns the batch, launches children, reads results, reports progress.

Child thread

Runs one task and returns one structured result.

Worktree

Isolated workspace for that child task.

Local state

Only enough to recover: run.json and results.jsonl.

Saved Files

.meta-skill/runs/<run-id>/ run.json results.jsonl

What Files Do

run.json

The batch ledger: task list, child thread ids, worktree paths, statuses, concurrency.

results.jsonl

Append-only stream: one completed, blocked, or errored result per child task.

What We Are Not Building In V1

No CLI

The Codex parent skill is the runner.

No comparison layer

Thirty tasks can run without ranking candidates.

No per-child archive

Threads and worktrees are the evidence; files are recovery state.

No export package

Later work, after the runner works.
