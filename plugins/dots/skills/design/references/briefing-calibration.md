# Briefing Calibration

Read this when forming the brief for a new surface, vague feature, standalone
redesign, substantial visual change, or UI-generation prompt. Skip it for a
small UI fix inside an existing design system.

## Brief Allocation

Spend brief detail where it gives the designer or model executable structure:

- layout skeleton: first viewport architecture, section order, grid, rails,
  sidebars, bands, lists, tables, canvases, or frames
- spacing rhythm: density, gutters, section gaps, padding, vertical tempo, and
  responsive collapse
- typography direction: display/body/UI roles, family class, weight, case,
  scale, line length, and chrome text treatment
- references: what each reference contributes, such as composition, type
  attitude, image treatment, interaction pattern, or density
- subject-specific material: real artifacts, data shapes, media, surfaces,
  rituals, instruments, environments, and vocabulary
- anti-patterns: visual tropes, component shapes, palettes, effects, or layout
  formulas the surface must avoid
- acceptance checks: what would make the result product-specific and
  client-ready for the stated context

Keep palette concise unless brand fidelity, accessibility, or a source target
requires exact values. Keep copy concrete, but do not let page-copy volume
replace design direction.

## Creative-Director Handoff

For a substantial surface, the brief should answer these in compact form:

1. Audience and job: who the surface serves and the primary action or decision.
2. Category stance: norms to respect, bend, or reject.
3. References: 2-4 anchors and the specific quality to borrow from each.
4. Type: display, body, UI chrome, weights, case, density, and line behavior.
5. Spacing and grid: container widths, columns, gutters, section cadence, and
   density.
6. Composition skeleton: first viewport, major regions, section rhythm, and
   responsive continuation.
7. Imagery and material: subject-specific assets, crops, media treatment,
   texture, icon language, and component motifs.
8. Motion: timing, easing, state changes, and moments that should stay still.
9. Avoid: explicit patterns that would make the result generic or off-brief.
10. Checks: what must be true before the surface is acceptable.

## Convert Mood Into Constraints

Do not leave adjectives to do the work alone. Translate them into visible
decisions:

- `sophisticated` -> editorial type pairing, restrained chrome, precise
  spacing, quiet dividers, no novelty gradients
- `tactile` -> visible paper, fabric, metal, ceramic, glass, print, grain, or
  physical controls from the subject's world
- `energetic` -> tighter rhythm, sharper contrast, asymmetric crops, faster
  transitions, stronger action hierarchy
- `calm` -> fewer competing tiers, longer line heights, softer motion, larger
  pauses between groups

Useful prompts name the type, grid, component shape, imagery, motion, and avoid
list. Weak prompts ask for a feeling and let the model choose every surface.

## Specificity Check

Before concepting, ask: what would make this impossible to swap into another
product with only copy and color changes?

If the answer is only a palette, slogan, or broad mood, strengthen the brief
with subject-specific imagery, data, materials, type behavior, spacing rhythm,
component motifs, and anti-patterns. Generic polish is not enough for a
product, brand, portfolio, or client-ready concept.

## Iteration

When revising a generated concept or prompt, preserve what works and give
targeted deltas. Prefer corrections like `increase gutter rhythm`, `replace
repeated grayscale portraits with varied project imagery`, `remove rounded pill
CTAs`, or `make table chrome denser` over broad resets such as `make it better`
or `rethink the design`.
