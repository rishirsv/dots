# Exhibits and tables

This playbook helps you choose between narrative, tables, and charts, and standardizes how exhibits are presented.

## Table vs chart vs narrative

Use **tables** when the reader needs:
- Exact values (tie-outs, bridges, schedules)
- Auditability (reported vs adjusted)
- Multi-line item financial statements

Use **charts** when the reader needs:
- Pattern recognition (trend, seasonality, mix)
- Comparison across categories quickly

Use **narrative** when:
- The point is qualitative (policy assessment, process gaps, risk description)
- Data is not available yet (but clearly list the data request)

## Exhibit minimums

Every exhibit must include:
- Exhibit number and title
- Period covered (e.g., FY2023–FY2025, LTM to MM/DD/YYYY)
- Units (e.g., $m, %, bps)
- Source/basis line

Example caption:
> Exhibit 4: Reported to Adjusted EBITDA Bridge (FY2025, $m)  
> Source: Management adjustment schedule and trial balance extract.

## Formatting standards for financial tables

Adopt the bundled financial modeling formatting conventions where applicable:
- **Negatives**: Use parentheses, not minus signs.
- **Units row**: Include a units row below headers (e.g., $m, %, x).
- **Totals/subtotals**: Bold totals and key subtotals to separate them from line items.

If you include a multi-period table, visually separate periods (e.g., historical vs LTM) in the layout.

## Adjustment bridge presentation

When presenting adjustments, prefer a table structure that supports auditability:

| Adjustment type | Description | Period | Impact ($m) | Recurring? | Basis |
|---|---|---:|---:|---|---|

Rules:
- Keep descriptions short; move detailed rationale to the narrative.
- If the basis is uncertain, label it and add an open item.

## Working capital presentation

Use a definition table first:
- WC definition, included accounts, excluded accounts, rationale.

Then show normalization:
- Actual vs normalized by month/quarter (if available)
- Or by FY/LTM with a clear caveat that seasonality may be missed

## Net debt presentation

Provide a schedule:
- Debt instruments, balances, interest rates (if known), maturity, comments.
- Separately list “debt-like” items reviewed and conclusions.

## Charts in DOCX

If you have numeric data and need charts:
- Generate chart images (PNG/JPG) first.
- Embed them in the DOCX near the relevant text (avoid orphaned charts with no interpretation).
- Always include captions with units, period, and source.

If you cannot generate charts, do not describe “a chart would show…” as a substitute; use a table and narrative instead.
